# План: Генерация синтетических данных для улучшения ITN

**Branch:** main
**Date:** 2026-06-05 22:12:06 MSK

## Мотивация

Текущее решение (97.6% accuracy) упирается в 12 ошибок на calibration.f.
Все ошибки — это паттерны, которых нет или мало в калибровочных данных.
Синтетические данные позволят:
1. Расширить словарь ASR-ошибок (Levenshtein-вариации)
2. Улучшить парсер редких группировок
3. Покрыть слитные написания (дветысячи, двестипятьсот)
4. Возможно, перейти к ML-подходу (seq2seq)

## Анализ текущих ошибок

| Тип ошибки | Кол-во | Описание |
|------------|--------|----------|
| Неверная группировка | 8 | слитные написания, неверное определение границ |
| Пропущенное число | 2 | "тыщ" без предшествующего числа |
| Лишнее число | 2 | "дваста" = 200 (но GT оставляет как есть) |

### 12 конкретных ошибок (для генерации контрпримеров)

1. `тыща четыреста` -> 1400 (слитное "тыща" не в словаре)
2. `дветысячи пятьсот` -> 2500 (слитное "дветысячи" было, но парсер не умножает)
3. `дваста` -> "дваста" (оставить, не нормализовать — расхождение с "двеси")
4. `шестьдесят с чем то тыщ` -> "60 с чем то тыщ" ("тыщ" одиноко)
5. `выше тыщ рублей` -> "выше тыщ рублей" ("тыщ" как существительное)
6. `двадцати тысячам` -> "20 тысячам" (нужно "20000")
7. `двести восемьдесят четвёртый` -> 284 (порядковое не просуммировалось)
8. `семьдесят миллионов два миллиона` -> "70000000 2000000" (два блока)
9. `на четырнадцатое число` -> "14 число" (порядковое не нормализовано)
10. `двести два два` -> "200 2 2" (перечисление, не сумма)
11. `двеси триста пятьдесят тысяч` -> 235000 (ASR-склейка)
12. `пять с половиной тыщ` -> "5 с половиной тыщ" ("тыщ" оставить)

## Архитектура генерации

```
scripts/generate_synthetic.py
  ├── load_templates()        — загрузка контекстных фраз из calibration.f
  ├── generate_clean()        — генерация чистых пар (число -> слова)
  ├── apply_asr_noise()       — применение ASR-искажений
  ├── apply_grouping()        — создание примеров суммы и перечисления
  ├── apply_splitting()       — создание слитных написаний
  └── save_dataset()          — сохранение в .feather
```

## Компоненты генерации

### 1. Базовые числа (clean)
Из числа -> словесная форма, используя `num2words` или наш `lexicon`:

| Разряд | Пример | Доля |
|--------|--------|------|
| 1-digit | 5 -> пять | 24% |
| 2-digit | 37 -> тридцать семь | 33% |
| 3-digit | 150 -> сто пятьдесят | 34% |
| 4-digit | 2500 -> две тысячи пятьсот | 5% |
| 5+ digit | 50000 -> пятьдесят тысяч | 4% |

### 2. ASR-шум (apply_asr_noise)
Типы искажений, применяемые к clean-форме:

| Тип | Пример | Вероятность |
|-----|--------|-------------|
| Замена гласных | пятьдесят -> петьсят | 30% |
| Пропуск букв | тысяча -> тыща | 20% |
| Цоканье | пятнадцать -> пятнадцить | 15% |
| Склейка слов | двадцать одно -> двадцатьодно | 15% |
| Редукция | девяносто -> дивяносто | 10% |
| Комбинация | несколько подряд | 10% |

ASR-шум применяется с вероятностью 60-70%, остальные 30% остаются чистыми.

### 3. Группировка (apply_grouping)
Контролируемое создание:

**Сумма (60% группируемых)**:
- сотня + десяток: `двести пятьдесят -> 250`
- тысячи + сотни: `три тысячи четыреста -> 3400`
- тысячи + сотни + десятки: `пять тысяч шестьсот семьдесят -> 5670`
- десятки тысяч: `двадцать пять тысяч -> 25000`
- сложные: `сто пятьдесят тысяч двести -> 150200`

**Перечисление (40% группируемых)**:
- повтор сотен: `двести триста -> 200 300`
- повтор десятков: `двадцать тридцать -> 20 30`
- перечисление единиц: `пять шесть семь -> 5 6 7`
- смешанное: `триста четыреста пятьсот -> 300 400 500`

### 4. Слитные написания (apply_splitting)
```
дветысячи, тристатысяч, пятьсотмиллионов
двестипятьдесят, тристашестьдесят
```
Из существующих раздельных форм склеиваем 30% обратно.

### 5. Контекстные шаблоны (load_templates)
Из calibration.f извлекаем контекстные фразы-обёртки:
```
"бюджет где то {число} рублей"
"примерно {число} показов"
"около {число} процентов"
"на {число} дней"
"в размере {число} рублей"
"суммарно от {число1} до {число2}"
```

### 6. Порядковые числительные
```
первый -> 1, второго -> 2, тринадцатого -> 13
двадцать первый -> 21, сто пятьдесят второй -> 152
```

## Известные ограничения

1. Wikipedia API не отвечает — нужен прямой HTTP парсинг
2. RSS даёт только заголовки (~80 предложений)

4. Нет Levenshtein-вариаций для неизвестных слов

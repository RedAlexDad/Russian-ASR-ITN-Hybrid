## Problems encountered

1. **FP16 даёт NaN градиенты на GTX 1650 Ti** — автоматическое включение fp16 убивало модель (loss=0.0, accuracy=0%)
2. **Отсутствует typeguard** — ошибка `generate-parameter-library-py` при установке
3. **ROS-плагины pytest** — `ament_*`, `launch_testing`, `launch_ros` конфликтовали с запуском тестов
4. **inputs не на device** — `generate_predictions` не переносил входные тензоры на GPU
5. **transformers 5.x несовместим с torch 2.6** — требует `torch>=2.7` и `float8_e8m0fnu`
6. **MLflow без проверки сервера** — долгий HTTP таймаут при недоступном сервере
7. **Токенизация в `__getitem__`** — каждый сэмпл токенизировался заново при каждой эпохе

## How they were solved

1. FP16 сделан опциональным флагом `--fp16`, по умолчанию `False`
2. `typeguard>=4.0` добавлен в `requirements.txt`
3. `pytest.ini` с `-p no:launch_testing -p no:launch_ros -p no:ament_*`
4. Добавлена строка `inputs = {k: v.to(device) for k, v in inputs.items()}`
5. Добавлено ограничение `transformers<5.0`
6. `socket.connect_ex` с таймаутом 0.5с перед запуском MLflow run
7. Пред-токенизация в `ITNDataset.__init__` — сохранение input_ids/labels в списках

## Problems encountered

- Wikipedia API в первые разы не отвечал — решил повторными попытками и user-agent
- RSS даёт только заголовки — пришлось догружать полные статьи по ссылкам
- Числа в Wikipedia часто привязаны к пунктуации: 1712—1733, 4.14, 17-09-2016
  → сложно заменять на словесную форму. Решение: нормализация пробелов вокруг пунктуации
- "Первый", "Вторая" — ordinals в реальном тексте. Парсер их нормализует (верно),
  но GT их не учитывает → accuracy 83%

## How they were solved

- Нормализация пробелов вокруг пунктуации перед заменой чисел
- Регулярка с негативным lookahead для чисел, привязанных к символам

## Notes for the future

- Реальные данные полезны для проверки распределения разрядов, не для accuracy
- Основная ценность: 4-digit числа (годы) — 33.9% против 5% в calibration
- Для точной валидации ASR нужна разметка ordinals в GT

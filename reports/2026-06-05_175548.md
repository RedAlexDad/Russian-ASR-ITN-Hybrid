## Мотивация

Текущий `solution.py` (412 строк) — монолит: словари, парсер, нормализатор и CLI
в одном файле. Это затрудняет поддержку, тестирование и масштабирование.

## Целевая архитектура

```
.
├── main.py                  # точка входа, вызывает src.cli
├── Makefile                 # help, run, evaluate, test, clean (цветной вывод)
├── src/
│   ├── __init__.py
│   ├── dicts/
│   │   ├── __init__.py
│   │   ├── units.py         # единицы, 10-19, десятки, собирательные
│   │   ├── hundreds.py      # сотни 100-900
│   │   ├── thousands.py     # тысячи/миллионы/миллиарды + MULTIPLIERS
│   │   ├── ordinals.py      # порядковые числительные
│   │   └── asr_errors.py    # ASR-ошибки → каноническая форма
│   ├── lexicon.py           # сборка NUMERAL_DICT, lookup_word/ordinal_value
│   ├── parser.py            # parse_number_group — логика сумма/перечисление
│   ├── normalizer.py        # normalize_text — обход токенов, групп, сборка
│   └── cli.py               # argparse: run, evaluate, list-errors
└── solution.py → удалить
```

## Схема связей

```
main.py
  └─ src.cli.main()
       ├─ src.normalizer.normalize_text()
       │    └─ src.parser.parse_number_group()
       │         └─ src.lexicon.lookup_word()
       │              ├─ src.dicts.units
       │              ├─ src.dicts.hundreds
       │              ├─ src.dicts.thousands  (+ MULTIPLIERS)
       │              ├─ src.dicts.ordinals
       │              └─ src.dicts.asr_errors
       └─ src.lexicon (для ordinals)
```

Каждый модуль имеет единственную ответственность:
- **dicts/*** — только данные (словари-константы)
- **lexicon.py** — построение общего словаря + функции поиска
- **parser.py** — чистая функция группировки токенов
- **normalizer.py** — обход текста, выделение групп, вызов parser
- **cli.py** — только CLI, никакой логики нормализации

## CLI-команды

```
python main.py run <input.f> [-o <output.f>]    # нормализация + сохранение
python main.py evaluate <calibration.f>          # accuracy на calibration
python main.py errors <calibration.f> [-n 15]    # показать N ошибок
python main.py --help                            # справка
```

## Makefile-команды (без команды → help)

| Цель       | Описание                              | Цвет вывода |
|------------|---------------------------------------|-------------|
| `help`     | Список целей (умолчание)              | cyan        |
| `run`      | `make run INPUT=data/test.f`          | green       |
| `evaluate` | `make evaluate INPUT=data/calibration.f` | yellow   |
| `errors`   | Показать первые 15 ошибок             | red         |
| `test`     | `python -m pytest` (если будут тесты) | blue        |
| `clean`    | Удалить __pycache__, .pytest_cache    | magenta     |

Цветовые префиксы в Makefile:
```
🔴 [ERROR]   — красный   (неудача, ошибка)
🟡 [WARN]    — жёлтый    (предупреждение)
🟢 [OK]      — зелёный   (успех)
🔵 [INFO]    — синий     (информация/замечание)
```

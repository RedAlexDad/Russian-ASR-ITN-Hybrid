## Рекомендации по дальнейшему улучшению

### 1. Regex-посткоррекция ASR-ошибок (оставшиеся 2 ошибки)

Добавить в `normalizer.py` или отдельный слой post-processing, который исправляет
известные ASR-паттерны ДО передачи в парсер:

```python
# "двеси" перед сотнями → "двести" + пересчёт числа
# "триста" в контексте "двести … тысяч" → "тридцать"
_ASR_CORRECTIONS = [
    (r'\bдвеси\s+(триста)', 'двести тридцать'),
    (r'\bдвеси\s+(четыреста)', 'двести сорок'),
    (r'\bдвеси\s+(пятьсот)', 'двести пятьдесят'),
]
```

Это исправит ошибку #11 ("двеси триста пятьдесят тысяч" → 235000).
**Риск:** ложные срабатывания, если "двеси триста" реально означает 200 и 300.

### 2. Levenshtein / Damerau-Levenshtein для ordinal-словаря

`is_ordinal_word()` сейчас работает через точное совпадение по `ORDINAL_SET`.
Добавить fallback на нечёткое сравнение для слов с ё/е, ь/ъ и типичными
опечатками.

Использовать `from Levenshtein import distance` (pip install python-Levenshtein)
или встроенный `difflib.get_close_matches`.

### 3. Динамическое расширение словарей

При запуске сканировать входные данные на слова с числовыми корнями,
отсутствующие в словарях. Для каждого не найденного слова проверять:
- Есть ли оно в ASR_ERRORS (обратное направление)?
- Содержит ли числовой корень из ORDINALS / NUMERAL_DICT?
- Похоже ли на известное слово через Levenshtein (dist <= 2)?

### 4. Контекстная эвристика для "дваста"

Сейчас "дваста" парсер конвертирует в 200. Проблема: GT сохраняет ASR-ошибку.
Решение: если "дваста" используется как числительное (согласуется с существ.
в роде/числе) → 200. Если после "дваста" идёт не-числительное → оставить как текст.

### 5. Mystem / pymorphy2 для определения частей речи

Подключить `pymorphy2` или `mystem` для:
- Определения, является ли слово числительным, даже если его нет в словаре
- Снятия омонимии (например, "сто" → 100 vs "сто" — глагол "стоять")
- Определения падежа для правильного согласования ("двадцати" → 20 в род.п.)

Установка: `pip install pymorphy2`
Словари: `pip install pymorphy2-dicts-ru`

### 6. Завершение T5 noisy training

Noisy training прерван на epoch 2 (eval_loss=1.47). Дозапустить:
```bash
make train-noisy MODEL_PATH=models/ruT5-itn EPOCHS=10
```
После дообучения включить T5 fallback:
```bash
HYBRID_USE_T5=1 make evaluate
```

### 7. n-gram как correction model (не confidence)

Вместо confidence: n-gram для rescoring альтернатив.
Генерировать N-best candidates через ruT5 с `num_beams=N` (сейчас 2),
ранжировать по CharNGram perplexity, брать лучший.

**Сейчас:** n-gram train=ground_truth (11.7 символов), infer=ASR (150+).
Для correction: n-gram train=правильные пары (input→output), infer=output кандидаты.
Длины будут совпадать (output ~11-30 символов).

# Словари и классификация числительных

Проект содержит два подхода к поиску числительных: **legacy lookup** (через `lexicon.py`)
и **TokenClass классификатор** (через `token_classifier.py`).

---

# Legacy: lexicon.py

## Структура словарей

Все словари расположены в `src/dicts/` и имеют единый формат:

```python
слово: (числовое_значение, порядок_величины)
```

### `units.py` — единицы, 10-19, десятки

| Раздел        | mag | Примеры                                     |
| ------------- | --- | ------------------------------------------- |
| Единицы 0-9   | 0   | `один: (1,0)`, `пять: (5,0)`                |
| Числа 10-19   | 1   | `одиннадцать: (11,1)`, `пятнадцать: (15,1)` |
| Десятки 20-90 | 2   | `двадцать: (20,2)`, `пятьдесят: (50,2)`     |
| Собирательные | 0   | `двое: (2,0)`, `трое: (3,0)`                |

### `hundreds.py` — сотни

| mag | Примеры                           |
| --- | --------------------------------- |
| 3   | `сто: (100,3)`, `двести: (200,3)` |

Включает ASR-искажения: `двеси: (200,3)`, `дваста: (200,3)`.

### `thousands.py` — тысячи, миллионы, миллиарды (умножители)

| Раздел    | mag | Примеры                             |
| --------- | --- | ----------------------------------- |
| Тысячи    | 4   | `тысяча: (1000,4)`, `тыщ: (1000,4)` |
| Миллионы  | 5   | `миллион: (1000000,5)`              |
| Миллиарды | 6   | `миллиард: (1000000000,6)`          |

### `ordinals.py` — порядковые числительные

Формат: `слово: "цифровое_значение"`. ~151 форма (включая ё).

```python
'первый': '1', 'второго': '2', 'тринадцатое': '13'
```

## Функции поиска

### `lookup_word(word)`

Точное совпадение в словаре. Возвращает `(val, mag, is_mult)` или `None`.

### `is_ordinal_word(word)` / `ordinal_value(word)`

Проверка и получение значения порядкового числительного.

## ASR-ошибки (`src/dicts/asr_errors.py`)

Словарь частых искажений из анализа calibration.f:

```python
'двеси': 'двести',          # 44+ вхождения
'тыщ': 'тысяч',             # 14 вхождений
'петьсят': 'пятьдесят',
'пятнадцить': 'пятнадцать',
```

# Новый: token_classifier.py

Root-based классификатор для sequence parser. Заменяет пайплайн
`lookup_word + is_ordinal_word + _cardinal_from_root`.

## TokenClass

```python
@dataclass
class TokenClass:
    value: int        # числовое значение
    mag: int          # разряд (0..6)
    subtype: str      # cardinal / ordinal / multiplier / fused / vague
    raw: str          # исходное слово
    confidence: float = 1.0
```

## `classify(word, prev_tokens=None)`

Пайплайн классификации:

1. **Vague контекст** — "тыщ"/"тыща" в бытовом контексте → subtype=vague
2. **Точное совпадение** в `NUMERAL_DICT` (O(1))
3. **Ordinal** — точное совпадение в `ORDINAL_SET`
4. **ASR-ошибки** — `ASR_ERRORS` → каноническая форма
5. **Root-based regex** — поиск по корням (длинные раньше коротких):

   ```python
   ("пятидесят", 50, 2, False),  # перед ("десят", ...)
   ("восемьдесят", 80, 2, False),
   ("тысяч", 1000, 4, True),
   ("двадцат", 20, 2, False),
   ```

   Покрывает ASR-искажения: `пятисят`, `семисят`, `восемдесят`, `двацать` и т.д.

6. **Ordinal suffix** — окончания "ый", "ой", "ая", "ое", "ые", "ых", "ым", "ого"
7. **Fused compound** — разбивка склеек:

   ```python
   "дветысячи" → ["две", "тысячи"] → TokenClass(2,0) + TokenClass(1000,4)
   "тристапятьдесят" → ["триста", "пятьдесят"] → TokenClass(300,3) + TokenClass(50,2)
   ```

Возвращает `list[TokenClass] | None`.

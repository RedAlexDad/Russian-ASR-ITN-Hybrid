# Парсер числовых последовательностей

Ядро решения: различение **суммы** (группировка) и **перечисления** (надиктовка).

## Разряды чисел (magnitude)

Каждое слово-числительное относится к одному из разрядов.
Разряд показывает, десятки это, сотни или тысячи.

| Разряд   | mag | Примеры слов                         |
| -------- | --- | ------------------------------------ |
| Единицы  | 0   | `один`, `пять`, `девять`             |
| 10-19    | 1   | `одиннадцать`, `пятнадцать`          |
| Десятки  | 2   | `двадцать`, `пятьдесят`, `девяносто` |
| Сотни    | 3   | `сто`, `двести`, `триста`            |
| Тысячи   | 4   | `тысяча` — **умножитель**            |
| Миллионы | 5   | `миллион` — **умножитель**           |

**Умножители** работают иначе, чем остальные разряды. Они умножают число,
которое стоит перед ними: `пять × тысяч = 5000`.

## Главное правило

Парсер смотрит на РАЗРЯДЫ (mag) соседних слов:

```
Разряд понижается (3 → 2 → 0) → СУММА:     двести пятьдесят = 200 + 50 = 250
Разряд не меняется (3 → 3)     → ПЕРЕЧИСЛЕНИЕ:  двести триста = 200, 300
Разряд повышается             → УМНОЖИТЕЛЬ:  пять тысяч = 5 × 1000 = 5000
```

## Примеры суммы (группировка)

Когда слова идут от старшего разряда к младшему — они складываются:

```
  "двести пятьдесят"
   сотни    десятки       ← разряд понижается (3 → 2)
   200   +   50      = 250

  "триста шестьдесят четыре"
   сотни   десятки  единицы  ← разряд понижается (3 → 2 → 0)
   300   +   60    +   4  = 364

  "две тысячи пятьсот"
   ед. × тысячи   сотни       ← умножитель (0→4), затем понижение (4→3)
   (2 × 1000) + 500 = 2500

  "двадцать один"
   десятки  единица           ← разряд понижается (2 → 0)
   20    +   1      = 21
```

## Примеры перечисления (надиктовка)

Когда слова одного и того же разряда — они не складываются:

```
  "двести триста"
   сотни   сотни          ← разряд ОДИНАКОВЫЙ (3 = 3)
   200     300            → два отдельных числа

  "двадцать тридцать"
   десятки десятки        ← разряд ОДИНАКОВЫЙ (2 = 2)
   20      30             → два отдельных числа

  "пять шесть семь"
   ед.  ед.  ед.          ← разряд ОДИНАКОВЫЙ (0 = 0 = 0)
   5    6    7            → три отдельных числа

  "триста четыреста пятьсот"
   сотни  сотни  сотни    ← разряд ОДИНАКОВЫЙ
   300    400    500      → три отдельных числа
```

## Как работает умножитель

Тысячи и миллионы не складываются, а **умножают** предыдущее число:

```
  "пять тысяч"
   ед. × тысячи
   5 × 1000 = 5000

  "двести пятьдесят тысяч"
   сотни + десятки = 250, затем × тысяч
   (200 + 50) × 1000 = 250000

  "пять тысяч двести"
   (5 × 1000) + 200 = 5200
```

Два умножителя одного уровня подряд — начало нового числа:

```
  "семьдесят миллионов два миллиона"
   (70 × 1000000)  +  (2 × 1000000)  =  70000000 + 2000000
                                  ↓
                           70 000 000 и 2 000 000 — два числа
```

## Порядковые числительные

Порядковые (первый, второй, тринадцатый) ведут себя как единицы (mag=0).
Они участвуют в суммировании, если стоят после старшего разряда:

```
  "двести восемьдесят четвёртый"
   сотни   десятки  порядковое    ← разряд понижается (3 → 2 → 0)
   200   +   80    +    4     = 284
```

## Наглядная таблица

| Фраза                    | Разряды           | Что происходит           | Результат |
| ------------------------ | ----------------- | ------------------------ | --------- |
| `двести пятьдесят`       | сотни → десятки   | понижение → сумма        | 250       |
| `двести триста`          | сотни → сотни     | одинаково → перечисление | 200 300   |
| `двадцать три`           | десятки → единицы | понижение → сумма        | 23        |
| `двадцать тридцать`      | десятки → десятки | одинаково → перечисление | 20 30     |
| `пять тысяч`             | единицы → тысячи  | умножитель               | 5000      |
| `пять тысяч двести`      | ед×тыс → сотни    | умножение + сложение     | 5200      |
| `двести пятьдесят тысяч` | сот+дес → тыс     | умножение                | 250000    |

## Функция `parse_number_group(tokens_data)`

Вход: список `(val, mag, is_mult, is_ordinal)`.
Выход: список строк-чисел.

```
(200,3) + (50,2) + (1000,4)        → ['250000']
(200,3) + (300,3)                  → ['200', '300']
(2,0) + (1000,4) + (80,1) + (5,0)  → ['2085']
```

# Парсер числовых последовательностей

Ядро решения: различение **суммы** (группировка) и **перечисления** (надиктовка).

## Принцип

Каждое слово-числительное имеет **порядок величины (mag)**:

| mag | Разряд | Пример |
|-----|--------|--------|
| 0 | Единицы | `один`, `пять` |
| 1 | 10-19 | `одиннадцать` |
| 2 | Десятки 20+ | `двадцать`, `пятьдесят` |
| 3 | Сотни | `двести`, `триста` |
| 4 | Тысячи (умножитель) | `тысяча` |
| 5 | Миллионы (умножитель) | `миллион` |

Парсер обходит токены слева направо и применяет правило:

```
mag строго убывает → СУММА (группируем)
mag не убывает     → ПЕРЕЧИСЛЕНИЕ (разделяем)
```

## Примеры

### Сумма (mag убывает)

```
"двести пятьдесят" → 250
 mag:  3      2      — убывает → 200 + 50 = 250

"две тысячи пятьсот" → 2500
 mag:  0  4      3    — 0<4 (умножитель) → 4>3 (убывает) → 2*1000 + 500 = 2500
```

### Перечисление (mag не убывает)

```
"двести триста" → 200 300
 mag:  3      3    — не убывает → два отдельных числа

"двадцать тридцать" → 20 30
 mag:    2      2    — не убывает → два отдельных числа
```

### Порядковые

Порядковые (mag=0) участвуют в суммировании наравне с единицами:

```
"двести восемьдесят четвёртый" → 284
 mag:  3        2          0    — убывает → 200 + 80 + 4 = 284
```

## Умножители

`тысяча`, `миллион` — умножают накопленное значение:

```
"пять тысяч" = 5 * 1000 = 5000
"двести пятьдесят тысяч" = (200 + 50) * 1000 = 250000
"пять тысяч двести" = (5 * 1000) + 200 = 5200
```

Два умножителя одного уровня подряд — разделение на два числа:

```
"семьдесят миллионов два миллиона" → 70000000 2000000
```

## Функция `parse_number_group(tokens_data)`

Вход: список `(val, mag, is_mult, is_ordinal)`.
Выход: список строк-чисел.

```
(200,3) + (50,2) + (1000,4)        → ['250000']
(200,3) + (300,3)                  → ['200', '300']
(2,0) + (1000,4) + (80,1) + (5,0)  → ['2085']
```

# Парсер числовых последовательностей

Проект содержит два парсера: **current** (legacy, mag-based) и **sequence** (state machine над TokenClass).
Выбор через `--parser-type {current,sequence}`.

---

# Current parser (legacy)

Ядро первоначального решения: различение **суммы** (группировка) и **перечисления** (надиктовка).

## Разряды чисел (magnitude)

Каждое слово-числительное относится к одному из разрядов.

| Разряд   | mag | Примеры слов                         |
| -------- | --- | ------------------------------------ |
| Единицы  | 0   | `один`, `пять`, `девять`             |
| 10-19    | 1   | `одиннадцать`, `пятнадцать`          |
| Десятки  | 2   | `двадцать`, `пятьдесят`, `девяносто` |
| Сотни    | 3   | `сто`, `двести`, `триста`            |
| Тысячи   | 4   | `тысяча` — **умножитель**            |
| Миллионы | 5   | `миллион` — **умножитель**           |

**Умножители** умножают число перед ними: `пять × тысяч = 5000`.

## Главное правило

```
mag понижается (3 → 2 → 0) → СУММА:      двести пятьдесят = 200 + 50 = 250
mag не меняется (3 → 3)    → ПЕРЕЧИСЛЕНИЕ:  двести триста = 200, 300
mag повышается             → УМНОЖИТЕЛЬ:  пять тысяч = 5 × 1000 = 5000
```

## Функция `parse_number_group(tokens_data)`

Вход: список `(val, mag, is_mult, is_ordinal)`.
Выход: список строк-чисел.

```python
(200,3) + (50,2) + (1000,4)         → ['250000']
(200,3) + (300,3)                   → ['200', '300']
(2,0) + (1000,4) + (80,1) + (5,0)   → ['2085']
```

Accuracy: **99.80%** (499/500) на calibration.f.

# Sequence parser (новый)

State machine поверх `TokenClass` (из `token_classifier.py`).
Включает root-based regex, lookahead heuristic и ASR-препроцессинг.

## TokenClass

```python
@dataclass
class TokenClass:
    value: int           # числовое значение
    mag: int             # разряд (0=единицы, 1=10-19, 2=десятки, 3=сотни, 4=тысячи...)
    subtype: str         # cardinal / ordinal / multiplier / fused / vague
    raw: str             # исходное слово
    confidence: float = 1.0
```

## Классификация (`classify()`)

Пайплайн определения типа токена:

1. **Точное совпадение** в `NUMERAL_DICT` (O(1))
2. **ASR-ошибки** → каноническая форма → словарь
3. **Root-based regex** — поиск по корням для неизвестных форм (тыяче, трста)
4. **Ordinal suffix** — проверка на порядковое окончание
5. **Fused compound** — разбивка склеек (дветысячи → два + тысячи)

## State machine (`parse_sequence()`)

```
compound = 0   # накоплено после умножителей
current = 0    # текущее накопленное число (< 1000)

Для каждого TokenClass:
  Vague        → пропустить
  Multiplier   → compound += (current or 1) × val
  Cardinal     → mag < last_mag → current += val (сумма)
                 mag >= last_mag → flush → enum (новое число)
  Ordinal      → то же, что cardinal, результат — порядковое
  Fused        → как cardinal/multiplier
```

### Lookahead heuristic

Предотвращает ложное суммирование единиц в перечислениях:

```
"двести два два"
   сотни ед. ед.  → mag: 3 → 0 → 0
   без lookahead:  200 + 2 + 2 = 202  ← неверно
   с lookahead:    flush(200), start(2), start(2) → 200 2 2  ← верно

"двести двадцать два"
   сотни дес. ед.  → mag: 3 → 2 → 0
   только один mag=0 → sum: 200 + 20 + 2 = 222  ← верно
```

### Vague-контекст ("тыщ", "тыща")

Если предыдущие токены указывают на бытовой контекст, "тыщ" маркируется
как `vague` и **пропускается** (не ломает парсинг):

```
"ну где-то тыщ пять"
   vague пропущен → 5

"около пяти тыщ"
   multiplier → 5000
```

### ASR-препроцессинг (`_asr_preprocess()`)

Regex-замены до парсинга:

| Паттерн | Замена | Пример |
| ------- | ------ | ------ |
| `двеси` → `двести` | Исправление ASR | двеси пятьдесят → 250 |
| Склейки (две+тысячи) | Разделение | дветысячи → два тысячи |
| `(?<=\d)\s+(?=\d{3}\b)` | Удаление пробелов | 200 000 → 200000 |

### Fused compound split

Разбивает слитные ASR-формы:

```
дветысячи → [TokenClass(2,0,"cardinal"), TokenClass(1000,4,"multiplier")]
пятьсотдвадцать → [TokenClass(500,3,"cardinal"), TokenClass(20,2,"cardinal")]
```

## Сравнение

| Аспект              | Current parser           | Sequence parser               |
| ------------------- | ------------------------ | ----------------------------- |
| Классификация       | lookup_word() (dict)     | classify() (root regex + ASR) |
| Неизвестные слова   | None → пропуск           | root regex → fused split      |
| ASR-формы           | Только точное совпадение | Regex + ASR-словарь           |
| Lookahead           | Нет                      | Есть (mag=0/1 дублирование)  |
| Перечисления ед.    | Иногда сливает           | Корректно (lookahead)         |
| Vague контекст      | Нет                      | Есть (тыщ/тыща)               |
| Accuracy калибр.    | 99.80%                   | 99.80%                        |

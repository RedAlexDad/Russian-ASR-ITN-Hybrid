# Нормализатор текста

Главная функция `normalize_text()` — обход токенов, выделение числовых групп,
замена на цифры. Подробнее об архитектуре: [docs/architecture.md](architecture.md)

## Алгоритм

```
Исходный текст → разбивка по пробелам → обход токенов

Для каждого токена:
  ├── Если числительное (количественное или порядковое):
  │     └── Собрать все следующие числительные в группу
  │     └── Передать группу в parser.parse_number_group()
  │     └── Вставить цифры вместо слов
  │
  └── Если не числительное:
        └── Оставить как есть (не изменять)
```

## Требование метрики

Accuracy требует **полного совпадения** строки с эталоном.
Любое изменение не-числового токена считается ошибкой.
Поэтому нормализатор **никогда не трогает** слова, не распознанные
как числительные.

## Связи с другими модулями

```
normalize_text()
    ├── lookup_word()        → src/lexicon.py
    ├── is_ordinal_word()    → src/lexicon.py
    └── parse_number_group() → src/parser.py
```

## Результат

Accuracy на calibration.f: **97.6%** (488/500 строк).
12 ошибок — преимущественно проблемы группировки и слитные написания.

## Гибридный режим

При низкой уверенности парсера (<0.75) подключается ruT5-small.
